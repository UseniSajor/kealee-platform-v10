## Why Offer Software-Only Packages

Packages A–D include consulting, PM services, and human expertise that justify premium pricing. Many contractors already have operational know-how — they just lack the tools. Software-only captures this segment at self-serve pricing.

**Revenue impact:** Software-only subscribers generate **85–95% gross margins** vs. 40–60% for full-service. They also serve as an on-ramp to full-service upgrades when projects outgrow DIY management.

## Implementation Prompt

> **Copy and paste this prompt into Claude or Cursor to implement the software-only pricing into the Kealee Platform codebase.**

```
You are helping me implement software-only SaaS subscription packages for the Kealee Platform v10. The platform is a Next.js monorepo using Fastify, Prisma, Supabase, and Stripe. I need you to create the following:

**1. Stripe Product/Price Setup Script**
Create a script that provisions these 4 new software-only packages in Stripe with their pricing tiers:
- S1 Starter: $29/mo (1 project), $49/mo (2 projects), $79/mo (3 projects)
- S2 Builder: $149/mo (5 projects), $249/mo (7 projects), $349/mo (10 projects)
- S3 Pro: $599/mo (15 projects), $899/mo (20 projects), $1,299/mo (30 projects)
- S4 Enterprise: $1,999/mo (50 projects), $3,499/mo (75 projects), $4,999/mo (100+ projects)

Each Stripe product should include metadata for: max_projects, max_users, feature_tier (s1/s2/s3/s4), and package_type: "software_only".

**2. Prisma Schema Updates**
Add to the existing subscription model:
- packageType enum: SERVICE, SOFTWARE_ONLY
- softwareTier enum: S1, S2, S3, S4
- Update the Subscription model with these fields
- Add feature entitlement checking based on tier

**3. Feature Gating Middleware**
Create middleware that checks the user's software tier and gates access to features:
- S1 (8 features): Bid Tracker, Daily Reports, Punch List, Progress Reports, Mobile Field Access, Change Order Tracking, Contract Manager, Safety Manager
- S2 (20 features): All S1 + Owner Dashboard, Schedule Manager, Lien Waiver Workflow, Budget Reports, Scope Matrix, Permit Wizard, Inspection Scheduler, RFI Portal, Submittal Manager, COI Tracker, QC Inspections, Document Control
- S3 (35 features): All S2 + AIA Pay Applications, Retention Manager, Sub Prequalification, Look-Ahead Scheduler, Bid Broadcast, Cash Flow Dashboard, Job Cost Reports, Back-Charge Manager, Sub Ratings, Sub Pay App Review, AI Takeoff Analysis, Labor Analytics, As-Built Manager, Selection Manager, Warranty Portal
- S4 (50 features): All S3 + all remaining features including Integration Hub/API

**4. Pricing Page Component**
Create a React pricing page at /pricing/software that:
- Shows all 4 packages in a comparison grid
- Highlights the most popular tier (S2 Builder)
- Toggles between monthly and annual pricing (20% annual discount)
- Links to Stripe Checkout for each tier
- Includes a "Compare to Full-Service" section
- Mobile responsive with swipeable cards

**5. Usage Limits Enforcement**
Implement project and user limits per tier:
- Check max_projects before allowing new project creation
- Check max_users before allowing team member invites
- Show upgrade prompts when approaching limits (80% threshold)
- Soft-lock (warning) at limit, hard-lock at limit + 1

**6. Upgrade/Downgrade Flow**
- Allow self-serve upgrades within software tiers (S1 → S2 → S3 → S4)
- Prorate billing on upgrades via Stripe
- Show "Unlock Full-Service" CTA on software tier dashboards
- Allow downgrade at end of billing period

Use the existing Kealee Platform conventions: app.kealee.com for the main app, api.kealee.com for the Fastify backend, Prisma for ORM, Supabase for auth, Stripe for billing. Follow the existing code patterns in the monorepo.
```
